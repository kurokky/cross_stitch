Inksacpeでクロスステッチデータを作る方法

How to Create Cross Stitch Data with Inkscape

はじめに

Introduction

このページを見ている方の大半はInk/Stitchの存在を知っていると思います。
Ink/StitchはInkscapeの拡張機能で、刺繍データを作成するためのツールです。
しかし、私が刺繍ミシンを手に入れた2025年12月時点でInk/Stitchを使ったクロスステッチのデータを簡単に作成する方法について有益な情報を見つけることができませんでした。
GithubのIssuesを見ても2023年で止まったままです。
色々調べると、フランスの方が書いたAlyogauという記事がありましたが、手順が省略されているのと、なんとか再現しても工程が多く「もっと簡易にデータを作れれば良いのに…」と思いました。

ここでは、私が開発したInk/PixelとStitchHelperとInk/Stitchを使用してクロスステッチデータを作成する方法を説明します。

I assume that those viewing this page are aware of Ink/Stitch.
Ink/Stitch is an extension for Inkscape and a tool for creating embroidery data.
However, as of December 2025, when I acquired my embroidery machine, I couldn't find any useful information on how to easily create cross-stitch data using Ink/Stitch.
Looking at the Issues on Github, they stopped in 2023.
After much research, I found an article called Alyogau written by a French person, but the steps were omitted, and even when I managed to reproduce it, the process was extensive, making me think, "I wish I could create the data more simply..."

Here, I will explain how to create cross-stitch data using Ink/Pixel, StitchHelper, and Ink/Stitch, which I have developed.

なぜInk/PixelとStitchHelperを使うのか？

Why use Ink/Pixel and StitchHelper?

最初は「ユーザーがInkscape作ったPixelのデータを変換するだけで良いのでは？」と思っていたのですが、2007年から20年弱もInkscapeを使い続けている私でも「Inkscapeはドット絵を作るのには向いていない」と言わざるを得えません。
(私は、2023年にアイロンビーズ用のツールを開発したことがあったのですが、そのインターフェースと比較してもInkscapeでドット絵を作成するのは非常に手間がかかります。)

私ががやりたいのは、ドット絵を簡単に作成して、それをクロスステッチデータに変換することです。
そこで、私はInkscapeでドット絵を簡単に作成できるInk/Pixelという拡張機能を作りました。かなり無理矢理な実装にしているので専用ツールには叶いませんが、ある程度のデータであれば矩形ツールを駆使して作成するよりも遥かに簡単にドット絵を作成できると思います。

そして、StitchHelperはドット絵をクロスステッチデータに変換するためのInkscapeの拡張機能です。
この2つの拡張を使うことで、ようやくInk/Stitchで刺繍用のデータを作成することができるようになります。
※右にある画面キャプチャとページの最初にある写真は下糸が切れたのを放置してしまい、少し失敗しています。

At first, I thought, "Wouldn't it be enough to just convert the pixel data created by the user in Inkscape?", but even for me, who has been using Inkscape since 2007, Inkscape is not suitable for creating pixel art.
(I developed a tool for Perler beads in 2023, and compared to that interface, creating pixel art in Inkscape is very time-consuming.)

What I want to do is easily create pixel art and convert it into cross-stitch data.
Therefore, I created an extension called Ink/Pixel that allows you to easily create pixel art in Inkscape. Although the implementation is quite forced and cannot compete with dedicated tools, I believe it allows for much easier creation of pixel art compared to using the rectangle tool for a certain amount of data.

And StitchHelper is an Inkscape extension for converting pixel art into cross-stitch data.
By using these two extensions, you can finally create embroidery data with Ink/Stitch.
(The screenshot on the right and the photo at the beginning of the page are slightly flawed because I neglected to change the bobbin thread when it ran out.)

使い方

How to Use

まずは、以下の方法でやってみてください。(工程が多いように感じますが、慣れると簡単に作れます)

First, please try the following method. (It may seem like a lot of steps, but it becomes easy once you get used to it.)

  1. InkscapeにInk/PixelとStitchHelperをインストールします。
    (Ink/Stitchと同じでDLしたファイルを解凍してextensionフォルダの中に入れてInkscapeを起動してください)
  2. まずは、刺繍枠に適したDocumetntのサイズを設定します。
    エクステンション -> StitchHelper -> 刺繍枠を選ぶ から自分で使っている刺繍枠のサイズを選択します。
    （見つからない場合はCtrl+Shift+Dで変更してください）
  3. エクステンション -> Ink/Pixel ->ドットの大きさ: クロスステッチ用(布目数) -> 11カウント / 1インチ (2.3mm) を選択します。
    （まずは2.3mmからはじめましょう）
  4. すると、自動でGridが自動で表示されDocumentサイズに沿って2.3mmの正方形が作成されます。
    (この正方形は、クリックしやすいように白のグラデーションがデフォルトになっています)
  5. 自分の作りたいデザインを作ります。
    （くれぐれもPixelのサイズを変更したり、グループ化などは使わないようにしてください）
  6. デザインができたら刺繍データにしたいPixelを選択します。
    （通常は何か1つのPixelを選択してCtrl+Aの全て選択で大丈夫です）
  7. エクステンション -> StitchHelper -> ドット絵から刺繍パターンを作成する -> 色ごとにレイヤーを作成 をクリックします。
  8. すると、色ごとに変換されたデータがレイヤー別にできます。
    (元のPixelデータのレイヤーは非表示になります)
  9. 刺繍データにしたいレイヤーのデータだけを選択します。
    (今回はハートの赤い部分だけを選択して作ります)
  10. エクステンション -> Ink/Stitch -> Tools:Satin -> サテン自動ルーティング を選択します。
  11. するとこのようなデータができるはずです。(クリックして確認してください)
  12. できたデータを選択しエクステンション -> Ink/Stitch -> Tools:Stroke -> Satin to Stroke をクリックします。
  13. このような感じになれば、成功です。(先程と同じように見えますが、微妙に違います)
  14. エクステンション -> Ink/Stitch -> パラメーター でデータを確認します。

Tips: 糸の細さが#50ならば「3」か「4」#60なら「3」をrepeatsに値を入れることをお勧めします。

  1. Install Ink/Pixel and StitchHelper in Inkscape.
    (Similar to Ink/Stitch, unzip the downloaded file, place it in the extension folder, and then start Inkscape.)
  2. First, set the Document size suitable for your embroidery hoop.
    Go to Extension -> StitchHelper -> Embroidery Hoop and select the size of the embroidery hoop you use.
    (If you cannot find it, please change it by pressing Ctrl+Shift+D)
  3. Select Extension -> Ink/Pixel -> Pixel Size: Cross stitch (fabric count) -> 11spi (2.3mm).
    (Let's start with 2.3mm first.)
  4. Then, the grid will be displayed automatically, and a 2.3mm square will be created along the document size.
    (This square has a white gradient by default for ease of clicking.)
  5. You will create the design You want to make.
    (Please be careful not to change the pixel size or use grouping.)
  6. Once the design is ready, select the Pixels you want to convert into embroidery data.
    (Usually, selecting one Pixel and pressing Ctrl+A for "Select All" is sufficient)
  7. Click Extension -> StitchHelper -> Create the Pattern from Pixel -> By Separating layers for each color.
  8. Then, data converted for each color can be created layer by layer.
    The original Pixel data layer will be hidden)
  9. Select only the data of the layers you want to turn into embroidery data.
    (This time, we will select only the red part of the heart to create it.)
  10. Click Extension -> Ink/Stitch -> Tools:Satin -> Auto-Route Satin
  11. You should get data like this. (Please click the image to check.)
  12. After selecting the generated data, click Extension -> Ink/Stitch -> Tools:Stroke -> Satin to Stroke
  13. This is what success looks like. (It looks the same as before, but it's subtly different.)
  14. Click Extension -> Ink/Stitch -> Params to check the data.

Tips: If the thread thickness is #50, we recommend entering "4" or "3" for repeats, and if it is #60, enter "3".

FAQ

  • クロスステッチをミシンでやる必要ってある？
  • 私も最初は「必要ないのでは？」と思っていましたが、クロスステッチはマス目が数えやすい布を選ぶ必要があります。その為刺繍を入れたい素材そのものに直接刺繍を行う事が難しく多くの場合はアップリケにする必要があるのですが、ミシンを使うことでミシンが対応する素材であれば生地を選ばなくなります。
    また、私は首の疾患を抱えているので長時間下を向く作業をすることができません。ミシンを使うことで負担を減らすことができます。
  • どうしてInk/Stitchにcommitしないんですか？
  • 自分が開発している時に、どこまでがInk/Stitchで対応すべきか？が分からなかったからです。クロスステッチにはPixelデータを作ることから始まり、刺繍データに変換するまでに多くのステップがあります。
    そのため、Ink/Stitchに全てを詰め込むのは適切ではないと考えました。
    ただし、将来的にはInk/Stitchに取り込まれる可能性もあります。
  • どうしてInk/Pixelで作られるPixelは白のグラデーションがついているのですか？
  • 最初は透明で、borderなしの正方形を作成していましたが、この状態だとInkscapeではクリックしづらく非常にストレスを感じました。そこで、クリックしやすいように白のグラデーションをつけることにしました。
    刺繍データに変換する際にはこのグラデーションは無視されるので問題ありません。
  • Is it necessary to do cross-stitch with a sewing machine?
  • At first, I also thought, "Isn't this unnecessary?"
    However, for cross-stitch, you need to choose fabric where the grid is easy to count. Therefore, it is difficult to embroider directly onto the material you want to embroider, and in many cases, you need to use embroidered patch. However, by using a sewing machine, you can embroider on any material that the sewing machine supports, regardless of the fabric type.
    Also, I have a neck condition, so I cannot work while looking down for long periods. Using a sewing machine can reduce the burden on me.
  • Why aren't you committing to Ink/Stitch?
  • Because when I was developing it, I wasn't sure how much should be handled by Ink/Stitch. Cross-stitch involves many steps, starting from creating pixel data and converting it into embroidery data.
    Therefore, I thought it would not be appropriate to include everything in Ink/Stitch.
    However, there is a possibility that it might be incorporated into Ink/Stitch in the future.
  • Why do the pixels created by Ink/Pixel have a white gradient?
  • Initially, they were transparent and created as squares without borders, but in this state, it was difficult to click them in Inkscape, which was very frustrating.
    So, I decided to add a white gradient to make them easier to click.
    This gradient is ignored when converting to embroidery data, so there is no problem.
